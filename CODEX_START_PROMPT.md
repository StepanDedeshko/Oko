# Стартовый промпт для Codex

Ты работаешь с проектом desktop-приложения **Око** версии `0.2 [pre-release]`.

Стек:
- Python 3;
- PySide6;
- QtWebEngine;
- Linux/Ubuntu/Cinnamon как основная целевая система.

Назначение:
Око — панель дежурного мониторинга Zabbix/ОТРС. Приложение показывает графики Zabbix, страницы проблем, режим дежурства, позволяет привязывать задачу ОТРС и открывать заметки.

Перед изменениями:
1. Изучи `PROJECT_CONTEXT.md`.
2. Кратко опиши структуру проекта.
3. Назови файлы, которые планируешь менять.
4. Не меняй unrelated-код.

Обязательные правила:
1. Не ломать запуск через `run_oko.sh`.
2. Не удалять `config.json`.
3. Не удалять пользовательские credentials.
4. Не добавлять автоматический `sudo`.
5. После изменений выполнить:
   ```bash
   python3 -m py_compile main.py
   find app -name "*.py" -print0 | xargs -0 -n1 python3 -m py_compile
   ```
6. Если меняется интерфейс, сохранить текущую логику:
   - Главная открывается по клику на логотип/название приложения.
   - Главная не находится в списке продуктов.
   - Темы настраиваются на Главной.
   - Доступы Zabbix/ОТРС настраиваются на Главной.
7. Если меняется установщик:
   - не запускать `sudo` автоматически;
   - только показывать команду для администратора;
   - сохранять `config.json` при обновлении.

Начни с краткого анализа проекта и предложи план изменения.
