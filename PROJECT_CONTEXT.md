# Око

Версия: `0.2 [pre-release]`

Око — desktop-приложение дежурного мониторинга для Zabbix/ОТРС.

## Назначение

Приложение предназначено для дежурных специалистов, которым нужно быстро открывать графики Zabbix, проверять состояние сервисов, фиксировать дежурные проверки и работать с задачами ОТРС.

## Основные функции

- просмотр графиков Zabbix во встроенных WebView;
- настройка продуктов и страниц без ручного редактирования `config.json`;
- режим дежурства с напоминаниями;
- привязка задачи ОТРС к дежурству;
- открытие страницы заметки ОТРС;
- автоподстановка логина/пароля ОТРС;
- хранение доступов Zabbix;
- главная страница;
- темы оформления;
- логотипы тем;
- заставка запуска по текущей теме;
- простой установщик и обновление.

## Текущая логика интерфейса

- Название приложения: **Око**.
- Версия: **0.2 [pre-release]**.
- В верхней панели отображается логотип активной темы и название приложения.
- Главная страница открывается по клику на логотип или название приложения.
- Главная страница не должна быть пунктом выпадающего списка продуктов.
- Выпадающий список продуктов содержит только рабочие продукты/разделы.
- Темы настраиваются на Главной странице.
- Доступы Zabbix и ОТРС настраиваются на Главной странице во вкладке профиля.

## Важные файлы

- `main.py` — точка входа приложения.
- `app/main_window.py` — главное окно, верхняя панель, меню продуктов, переключение страниц.
- `app/home_config.py` — Главная страница, профиль, темы, продукты/страницы, заметки.
- `app/login_dialog.py` — окно авторизации.
- `app/duty_mode.py` — режим дежурства, ОТРС-задачи, заметки.
- `app/dashboard_widgets.py` — графики, страницы проблем, WebView.
- `app/theme.py` — темы оформления.
- `app/theme_logo.py` — загрузка логотипов тем.
- `app/splash.py` — заставка запуска.
- `app/credentials.py` — сохранение Zabbix-доступов.
- `app/screen_utils.py` — работа с мониторами.
- `app/app_info.py` — название, версия и описание приложения.
- `config.json` — основная конфигурация.
- `CHECK_SYSTEM_DEPS.sh` — проверка системных пакетов перед запуском.
- `run_oko.sh` — основной запуск.
- `INSTALL_OKO_GUI.sh` — графический установщик.
- `INSTALL_OKO.sh` — терминальный установщик.
- `UPDATE_OKO.sh` — обновление из нового архива.
- `ROLLBACK_OKO.sh` — откат на последнюю резервную копию.

## Системные зависимости Ubuntu

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip python3-tk libxcb-cursor0 libnss3 libxcomposite1 libxdamage1 libxrandr2 libgbm1 libasound2t64
```

На старых системах вместо `libasound2t64` может быть `libasound2`.

## Правила разработки

- Не удалять пользовательский `config.json`.
- Не удалять сохранённые credentials без явного запроса.
- Не ломать portable-запуск через `run_oko.sh`.
- Не возвращать автоматический `sudo` в запускатели.
- Если не хватает системных пакетов, показывать команду для администратора, но не запускать `sudo` автоматически.
- После изменений запускать проверку компиляции Python-файлов.
- Крупные изменения делать маленькими шагами.
- Изменения, требующие перезапуска, должны использовать общий механизм уведомления о перезапуске.
