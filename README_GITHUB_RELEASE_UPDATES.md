# Обновления через GitHub Releases

Этот проект можно обновлять прямо из приложения по URL архива.
Рекомендуемый способ — хранить архивы обновлений в **GitHub Releases**.

## Что создать на GitHub

1. Открой репозиторий на GitHub.
2. Перейди в раздел **Releases**.
3. Создай новый релиз с тегом (например, `v0.2.1`).
4. Приложи архив обновления как asset, например `update.zip`.

После этого прямая ссылка на обновление будет вида:

`https://github.com/<owner>/<repo>/releases/download/<tag>/update.zip`

Эту ссылку вставляй в поле **URL архива обновления** на вкладке **Главная → Обновление**.

## Требования к архиву

Внутри архива должны быть `main.py` и папка `app`.

## Важно

- `UPDATE_OKO.sh` делает backup перед обновлением.
- `config.json` сохраняется и возвращается после обновления.
- `sudo` не запускается автоматически.
