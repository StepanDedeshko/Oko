ЗАСТАВКА ДО ПОЛУЧЕНИЯ СЧЁТЧИКА ПРОБЛЕМ

Теперь загрузочная заставка может висеть не по фиксированному времени,
а пока не появится число проблем на странице Zabbix.

Логика:
1. Пользователь вводит логин/пароль.
2. Открывается основное окно Дежурки.
3. Поверх него показывается загрузочная заставка с видео/музыкой.
4. Дежурка загружает страницу "Проблемы".
5. Когда найдено число из .table-stats, заставка закрывается.
6. Если счётчик не найден за max_wait_ms, заставка закрывается по таймауту.

Настройка в config.json:

"loading_screen": {
  "enabled": true,
  "show_until_problem_counter_ready": true,
  "max_wait_ms": 30000,
  "show_after_login": false
}

max_wait_ms — максимум ожидания в миллисекундах.
30000 = 30 секунд.

Если Zabbix недоступен или не авторизовался,
заставка не будет висеть бесконечно.
