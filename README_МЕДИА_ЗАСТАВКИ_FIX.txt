ЕСЛИ ВИДЕО И МУЗЫКА НЕ ПОЯВЛЯЛИСЬ

Исправлено:

1. Теперь не обязательно называть файлы строго loading.mp4 и loading_music.mp3.
   Если auto_detect_media=true, приложение само найдёт первый видео/аудио файл в папке:
   assets/loading

2. Поддерживаемые расширения:
   Видео: mp4, webm, mov, mkv, avi
   Аудио: mp3, wav, ogg, flac, m4a

3. Добавлено минимальное время показа заставки:
   min_show_ms = 5000

   То есть даже если счётчик проблем нашёлся быстро,
   заставка не закроется мгновенно.

4. Обновлятор теперь сохраняет assets/loading,
   чтобы твои видео/музыка не затирались при обновлении.

5. На заставке теперь показывается, какой файл найден:
   Видео: имя_файла.mp4 | Музыка: имя_файла.mp3

Куда класть файлы:
   assets/loading/

Лучший вариант:
   assets/loading/loading.mp4
   assets/loading/loading_music.mp3

Если видео найдено, но не проигрывается:
скорее всего проблема в кодеке. Самый надёжный вариант:
MP4 H.264.
