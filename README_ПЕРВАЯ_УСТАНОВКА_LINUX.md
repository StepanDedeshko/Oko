# Первая установка Око в Linux

При первом запуске приложение создаёт `config.json` из шаблона `config.example.json`. В публичном шаблоне нет включённых Zabbix-серверов, потому что рабочие адреса, логины и секреты нельзя хранить в репозитории.

Если на экране входа показано состояние первичной настройки, выполните один из вариантов ниже.

## Вариант 1: импортировать готовый config.json

1. Нажмите **Импортировать config.json** в окне входа.
2. Выберите подготовленный JSON-файл конфигурации.
3. Дождитесь сообщения: `config.json импортирован. Перезапустите приложение.`
4. Перезапустите приложение.

Перед заменой текущего файла приложение сохраняет резервную копию рядом с ним в формате `config.json.before_import_YYYYMMDD_HHMMSS`.

## Вариант 2: открыть папку приложения и настроить вручную

1. Нажмите **Открыть папку приложения**.
2. Откройте `config.json` в текстовом редакторе.
3. Добавьте свои Zabbix-серверы в `zabbix_instances` и включите нужные записи через `"enabled": true`.
4. Сохраните файл и перезапустите приложение.

## Важно

- Не публикуйте рабочий `config.json` в Git.
- Не добавляйте в публичный репозиторий пароли, токены, внутренние URL или другие секреты.
- Если у вас уже есть рабочий `config.json` с включёнными Zabbix-серверами, обычный вход продолжит работать без дополнительных действий.
