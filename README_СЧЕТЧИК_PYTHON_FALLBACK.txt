СЧЁТЧИК ПРОБЛЕМ — УСИЛЕННЫЙ ПОИСК

Добавлен второй способ поиска счётчика.

Теперь порядок такой:
1. JavaScript ищет элемент .table-stats.
2. Если не нашёл — Python получает весь видимый текст страницы через toPlainText().
3. Если не нашёл — Python получает HTML страницы через toHtml().
4. В тексте/HTML ищется строка вида:
   Отображено с 1 до 100 из 294 найденных
5. Если строка найдена — берётся 294.

Это должно помочь, если Zabbix дорисовывает таблицу позже
или .table-stats недоступен в момент первого JS-поиска.

Если всё равно будет "н/д":
наведи мышь на плашку "Всего проблем: н/д".
В tooltip будет фрагмент текста страницы, который реально видит приложение.
По нему можно будет понять, видит ли приложение нужную строку вообще.
