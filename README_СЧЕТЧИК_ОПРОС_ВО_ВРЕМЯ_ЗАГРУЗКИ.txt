СЧЁТЧИК ПРОБЛЕМ — ОПРОС ВО ВРЕМЯ ЗАГРУЗКИ

Добавлено:
пока висит загрузочная заставка, приложение каждые 5 секунд
принудительно пытается получить число проблем из страницы Zabbix.

Зачем:
Zabbix может дорисовывать таблицу после loadFinished,
поэтому одноразовый поиск мог не успевать.

Настройки в config.json:

"loading_screen": {
  "show_until_problem_counter_ready": true,
  "max_wait_ms": 30000,
  "problem_counter_poll_interval_ms": 5000
}

Логика:
1. Открывается приложение.
2. Показывается загрузочная заставка.
3. Каждые 5 секунд приложение ищет .table-stats.
4. Если найдено:
   <div class="table-stats">Отображено с 1 до 100 из 294 найденных</div>
   берётся число 294.
5. Заставка закрывается.
6. Если за 30 секунд не найдено — заставка закрывается по таймауту.
